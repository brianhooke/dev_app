================================================================================================
SERVICES ORDER TEMPLATE
For consistent views.py documentation across all PROJECT TYPE apps
================================================================================================

This file defines the standard order for documenting functions in any PROJECT TYPE's views.py file.
When documenting views.py, list ALL services in this exact order, even if no functions exist for that service.

USAGE:
1. Copy the structure below
2. Replace {APP_NAME} with the actual app name (e.g., 'dashboard', 'core', 'claims_app')
3. For each service, list the functions that exist in that views.py file
4. If no functions exist for a service, still include the heading with "(none)" or leave empty

================================================================================================
STANDARD SERVICES ORDER:
================================================================================================

1. SELF (App-Specific Views)
   - These are views specific to the app itself (e.g., dashboard homepage, app landing pages)
   - Use format: "{APP_NAME} View:" or "{APP_NAME} Views:"
   
2. Contacts
   - Contact management, verification, CRUD operations
   - Use format: "Contacts Views:"

3. Bills
   - Invoice management, allocations, bill processing
   - Use format: "Bills Views:"

4. Quotes
   - Quote management, quote allocations, quote processing
   - Use format: "Quotes Views:"

5. POs (Purchase Orders)
   - Purchase order management, PO processing
   - Use format: "PO Views:"

6. Claims
   - Claims management, claim allocations, claim processing
   - Use format: "Claims Views:"

7. Xero
   - Xero API integration, OAuth, instance management
   - Use format: "Xero Views:"

================================================================================================
NUMBERING CONVENTION:
================================================================================================

- Number functions sequentially across ALL services (1, 2, 3, 4... not restarting for each service)
- Example:
  
  Dashboard View:
  1. dashboard_view - Main homepage
  
  Contacts Views:
  2. verify_contact_details - Save verified details
  3. pull_xero_contacts - Pull from Xero
  
  Bills Views:
  4. send_bill - Send to Xero
  5. archive_bill - Archive bill
